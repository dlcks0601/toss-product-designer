## 1. 문제 정의와 서사

**프레임: "조율은 계산이 아니라 배려다."**

기존 캘린더·조율 도구(겹쳐보기, When2meet, Doodle)의 전제는 "모두의 빈칸이 겹치는 곳을 찾으면 끝"이다. 그러나 실제 조율에서 고민하는 건 빈칸이 아니라 *이 시간이 그 사람에게 괜찮은가*다. 가능/불가라는 이진 답변이 사람의 사정을 지운다.

**발견한 문제 4겹 → 각각 해법 장치 1개:**

| # | 문제 | 해법 장치 |
|---|---|---|
| 1 | 캘린더는 시간을 보여주지만 사람을 읽어주지 않는다 (외근 요일·점심 리듬·정기 미팅은 캘린더 안에 이미 있는데 조율자가 머리로 기억) | 패턴 읽기: 프로필 피크 + 스캔 모먼트 + 이유 칩 |
| 2 | '모두가 되는 시간'은 자주 없는데 도구는 그 현실을 못 다룬다 ("앞 30분은 가능"이 "불가"로 뭉개짐) | 부분 참석 + 꼭/선택 구분 + 정직한 트레이드오프 카드 |
| 3 | 조율에는 기한이 있는데 도구는 시급함을 모른다 | 기한 일급 입력 + 결정 모먼트("기한을 지킬까요, 미룰까요?") |
| 4 | 잡은 뒤의 미안함(눈치)을 도구가 안 덜어준다 | 배려를 시스템이 표현: 양해 문구·10분 늦추기·50분 회의·화상 링크 |

뿌리 명제: **"일정은 혼자면 결정, 사람이 생기면 협상이 된다"** → 참석자를 추가하는 순간 폼이 변하는 모핑으로 이 명제 자체를 인터랙션으로 표현.

점심 문제의 재정의(자율 점심 11~15시 전제): 예측하지 말고 보장한다 —
- **리듬 읽기**: 개인별 평소 점심 창을 캘린더 패턴에서 추론. 그 직후 30분 내 시작 슬롯에 소프트 감점 + 칩("보통 1시쯤 점심을 먹어요 — 2시 시작은 나른할 수 있어요")
- **점심 보호**: 회의가 들어가도 전원에게 11~15시 사이 연속 60분 빈 구간이 남는지 검사. 미달 시 경고("점심 여유가 25분뿐이에요")
- 전사 고정 점심(13~14) 폐기

**언어 규율**: 사람에 대한 감정·요구 언어 금지("싫어해요" ❌). 항상 관찰된 패턴의 언어("매주 화 10시 정기 미팅이 있어요" ✅). 선호는 사람이 아니라 슬롯에 붙인다("점심 직후 시간이에요 · 10분 늦춰 시작할까요?").

**필수답변 3종 초안:**
- Q1(발견한 문제): 위 4겹 서사. "점심시간이 고정이라는 가정부터 의심했다"를 본질 발견의 증거로.
- Q2(어떻게 해결): "사람을 읽는 캘린더" — 4겹에 1:1 대응하는 4개 장치.
- Q3(왜 이 방식): 이 제품의 답은 반응하는 경험(조건 변경 → 즉시 재배열, 사람 언어 이유)이라 정지 화면으로는 표현 불가 → 실제 동작하는 웹앱(PC+모바일). 판단은 실물에서만 가능하고, 동작하는 레포는 그 자체가 스펙.

## 2. 제품 구조 — 두 여정 체제

제품명·로고: v1의 **toss calendar 워드마크 + 토스 심벌 그대로 사용**.

**홈(내 캘린더) = 분기점.** 시나리오 치트 메뉴 없음. 홈에 자연스럽게 존재하는 두 입구:
- 여정 A(보내기): `새 일정 만들기` CTA
- 여정 B(받기): 캘린더에 이미 도착해 있는 초대("민수님이 보낸 초대 · 응답 대기" 배지)

### 여정 A — 초대를 보내는 입장 (핵심 플로우)

**A1. 홈**: 내 주간 캘린더(내 일정 + 점심 등), 받은 초대 카드, 할 일 카드, CTA 하나. 드래그 없음.

**A2. 셋업 — 하나의 폼, 모핑**:
- 혼자: 제목·(선택)메모·날짜·시작~종료. 그대로 저장하면 개인 일정.
- 참석자 추가 순간 모핑(450ms 한 호흡): 날짜·시간 필드가 접히고 → "회의 길이" 칩(30분/1시간/1시간 30분, 기본 1시간) + **"언제까지 잡아야 하나요?" 기한 칩(이번 주 안에/다음 주까지/여유 있어요, 기본 다음 주까지)** 등장, CTA `일정 만들기` → `시간 찾아보기`.
- 참석자 리스트: 이름 + 꼭/선택 핀(**기본값 = 꼭 참석**). 아바타 탭 → **프로필 피크('다' 안)**: 헤더 없음(행 아래 인라인/PC 팝오버), 미니 주간 캘린더 블록(실제 일정, 기한 창 범위) + 맨 아래 각주 한 줄("🚗 목요일 외근이 잦은 편이에요"). 기본 접힘.
- 참석자 피커: 20명 조직 풀 검색·선택 (v1 구조 계승, 여기서도 아바타 탭 피크 가능).

**A3. 스캔 모먼트 (최초 1회만)**: `시간 찾아보기` 직후 ~1.2초 — 아바타 순차 점등 + "준호님의 외근 요일을 확인했어요 → 세훈님의 점심 리듬을 살폈어요" 문장 시퀀스 + 진행 바. 패턴 추출기의 실제 출력 사용(가짜 문장 금지). 이후 조건 변경엔 절대 재등장. reduced-motion 시 생략+aria-live.

**A4. 시간 찾기 (히어로 화면)** — 뷰포트별 무대:
- **모바일(기본)**: 추천 카드 리스트. 카드 = 시간 + 뱃지(추천/일부 아쉬움/주의) + 요약 1줄 + **미니 주간 로케이터**(5칸 스트립에 위치 점) + 탭하면 그 자리에서 이유 칩 확장. `시간표로 보기` 토글로 풀 그리드(보조).
- **PC**: 지도 앱 패턴 — 주간 캘린더 캔버스(후보 = 차분한 밴드 레이어, 팀 일정 = 무채색 블록·호버 시 소유자) + 우측 이유 카드 레일. **상호 하이라이트**: 카드 선택 → 밴드 글로우 / 밴드 탭 → 카드 포커스. 캔버스가 중반까지 아름답지 않으면 탭 구조로 후퇴(카드 재사용, 손실 없음).
- **이유 밀도 규칙**: 전원 가능은 한 줄로 합침("필수 4명 모두 편하게 참석할 수 있어요"). 줄은 긴장에만 — 부분 참석("하늘님은 앞 30분만 함께할 수 있어요 — 11시에 다른 회의") / 점심 리듬 / 점심 보호 / 외근+화상 대안 / 회의실. 점수 절대 비노출.
- 조건 변경(길이·기한·꼭/선택) → 즉시 FLIP 재정렬. 로딩 없음.
- 키보드 후보 순회(←→) 유지.

**A5. 결정 모먼트 (후보 전멸 시 1회)**: "이번 주까지는 모두가 편한 시간이 없어요" + 실제 시뮬레이션으로 열리는 것만 제안: ① 기한 미루기("후보 9개가 생겨요") ② 회의 30분으로 줄이기 ③ **필수 참석자 부분 참석 허락**("준호님이 앞 30분만 함께해도 괜찮다면 — 수 2:00가 열려요"). ③은 여기서만, 명시적 허락으로만.

**A6. 확정**: 선택 슬롯 요약 + 완화 토글(맥락 맞춰: 10분 늦춰 시작/50분 회의/부분 참석자 퇴장 5분 전 알림) + 회의실 목록(정원·점유 반영, **자동 선택 없음, `추천` 뱃지만**. 방 없으면 화상 옵션에 뱃지). CTA "이 시간으로 확정할게요".

**A7. 완료**: 체크 팝 + 확정 요약 + **"함께 챙긴 것"** 배려 카드(양해 문구·안건 순서 제안·화상 링크) + `참석자에게는 이렇게 보여요`(여정 B 다리) + `내 캘린더에서 보기`. **응답 도착 연출은 여기가 아니라 홈 토스트로**(완료 화면은 "초대를 보냈어요" 상태까지).

**A8. 홈 복귀 — 응답 도착**: 확정 후 홈으로 돌아오면 응답이 실시간 토스트로 도착(3초 준호 ✓ → 6초 서연 ✓ → 10초 하늘 "앞 30분 참석") + 캘린더 회의에 응답 배지 채워짐 + 알림 센터 적립.

### 여정 B — 초대를 받는 입장

**B1. 초대 뷰**: 홈의 응답 대기 초대 탭 → "민수님이 회원님 포함 5명과 잡은 1시간" + **나를 기준으로 재계산된 이유**("내 이야기: 금요일 마감 회의를 피했어요" — 내 행은 강조) — 주최자가 챙긴 배려가 수신자에게 보인다.
**B2. 응답**: `참석할게요` / `어려워요`(사유 칩 1회, 재조율 없음) → "민수님에게 전달했어요" → 내 캘린더에 회의가 자리 잡음.
선호 등록 온보딩 폐기(캘린더가 읽으니까).

### 알림 시스템

- 알림 센터: 응답·초대·확정 알림 모아보기. PC = 우하단 플로팅 벨 / 모바일 = 헤더 벨.
- 실시간 토스트: PC = 우하단 슬라이드업 스택(최대 3, 4초 소멸) / 모바일 = 상단 드롭(하단 CTA 충돌 방지).
- `notifications` 스토어 하나를 토스트 큐와 알림 센터가 공유.

### 심사자 첫 3분

- **웰컴 카드(첫 방문 1회)**: 캘린더 위에 얹히는 카드 한 장 — "동료 6명과 다음 주까지 1시간 회의를 잡아야 해요" + `회의 잡아보기`(6인 프리필 셋업 진입) / `천천히 둘러볼게요`.
- **할 일 카드(상시)**: 캘린더 상단 리마인더 "📌 다음 주까지: 팀 회의 잡기 · 6명" → 셋업 6인 프리필. 시나리오 메뉴의 대체품.

## 3. 엔진 계획 (v1 이식 + 감사 반영)

이식 원칙: 순수·결정적 코어(Date.now 무의존)와 완화 시뮬레이션 하네스, rankSlots 골격은 살린다. 감사가 찾은 결함 13건은 이식 중 수정 체크리스트로 처리.

**8단계 최소 리스크 경로** (감사 권고 채택):
1. 공용 유틸 추출(weekdayIndex·toISODate·overlaps 단일화) + 타임존 UTC 통일 — 행동 불변, 기존 테스트가 안전망
2. **점수↔카피 분리**: scoreSlot이 구조화된 `{code, delta, data}[]`를 반환, 한국어 문장은 별도 포매터 — 이유 칩 시스템·테스트 가능성의 기반
3. 기한 윈도우(R1): 기한 칩→days 빌더(주말 방어 포함), urgent 가중치 2지점 제거
4. 개인 점심 리듬 + 점심 보호(R4·R5): Attendee.lunchWindow(nullable), COMPANY_LUNCH 하드필터→소프트 페널티 전환, 11~15 최장 연속 빈 구간 규칙 신설, UI 상수 결합 해소
5. 부분 참석—선택(R2): 이진 판정→겹침 인지 {full|partial(앞/뒤 N분)|none} + 중간 점수, CandidateSlot 확장
6. 부분 참석—필수 허락제(R3): 완화 하네스에 새 patch kind. 선행 수정: 완화에 rooms 통과, findBottleneck 주최자 제외
7. 패턴 추출기(R6, insights.ts 신설): events에서 외근 요일 빈도·반복 회의(요일+시각)·점심 리듬·한가한 구간 역산. **프로필 피크 각주·스캔 모먼트 문장·이유 칩이 같은 추출기 공유**(일관성 계약)
8. 응답 타입(R8): 응답 타임라인, 'difficult' 정식화 — 엔진 무접촉

**결함 수정 체크리스트** (감사 §3): ①점수 스케일 정규화 — 선택 가능자 +10 무한 누적 편향 제거(20명 요구와 정면충돌) ②완화 시뮬에 rooms 전달 ③back-to-back 직전 방향 누락 버그 ④extend-days 라벨↔patch 불일치 ⑤top-5 절단 vs 결정 모먼트 전수검사 불일치(표시 후보 기준으로 통일) ⑥주말 방어 ⑦findBottleneck 주최자 제외 ⑧시간 매직넘버(720/840/990) 제거 — duration·개인 점심 기준 상대 계산으로 ⑨중복 유틸·포매터 정리 ⑩타임존 통일. 신규 규칙마다 테스트 추가, 미핀 구멍(직전 back-to-back, step≠30, 빈 프레임, 주말) 테스트 보강.

**데이터 로딩 원칙**: "뷰가 필요한 창 단위로" — 홈은 보고 있는 주, 스케줄러 계산은 기한 창, 피크는 논의 중인 창. 제품 데이터 자체는 자유 탐색 가능(기한으로 제품을 자르지 않는다).

## 4. 목데이터 각본 (src/data/world.ts)

원칙: **목데이터는 데모의 대본.** 결정적 — `ANCHOR_DATE = '2026-07-07'(화)` 고정, 이번 주 = 7/6(월)~7/10(금), 다음 주 = 7/13(월)~7/17(금), 여유 창 = ~7/24. `Date.now()` 금지. 20명 **전원 정품질** — 각자 근무 리듬·데모 기간 전체 일정·읽힐 패턴(외근/정기 미팅/점심 리듬/유연근무 다양 분배). 누구를 골라도 피크·스캔·이유 칩이 그 사람을 읽는다. 6인은 기본 출연진일 뿐 데이터 특권 없음.

| 인물 | 역할 | 체화하는 단서 | 발동 장면 |
|---|---|---|---|
| 이찬(나) | 주최자 | 점심 리듬 12:30 | 시스템이 주최자도 공평하게 읽음 |
| 박준호 | FE · 꼭 | 목요일 외근 잦음 + 매주 월 10시 스프린트 | 외근 칩+화상 대안, 결정 모먼트 부분 참석 |
| 이서연 | 데이터 · 꼭 | 화·수 오전 집중 + 오후 빡빡 | 점심 보호("여유 30분뿐") |
| 최민수 | 디자인 · 꼭 | 유연근무 10~19시 | 근무시간 교집합이 프레임 좌우 |
| 정하늘 | 마케팅 · 선택 | 화요일 외근 + 수 11시 회의 | 부분 참석("앞 30분만") |
| 오세훈 | BE · 선택 | 점심 리듬 13:00 + 일정 촘촘 | 점심 직후 칩 + "10분 늦추기" |

**장면 계약(테스트로 고정)**: S1 기본 6인+다음 주 → 후보 ~9개, 1위 수 10:00(하늘 부분 참석 포함) / S2 화 14:00 → 세훈 점심 칩+완화 / S3 목 11:00 → 준호 외근+서연 점심 보호 / S4 기한 "이번 주"→후보 0→결정 모먼트 3선택지 / S5 민수→나 초대 1건(여정 B) / S6 확정 후 응답 토스트 각본.

## 5. 모션·비주얼 시스템

### 모션 토큰 (토스 챌린지 페이지 실측값 채택 + Rally 원칙)

Rally 원칙: 이징은 bezier와 spring 두 계열만. 한 타겟엔 하나의 모션 규칙. 다중 타겟은 타임라인(스태거)로.

```css
--bezier-standard: cubic-bezier(0.4, 0, 0.2, 1);   /* 표준 트랜지션 */
--bezier-out:      cubic-bezier(0.25, 0.1, 0.25, 1);
--bezier-expo:     cubic-bezier(0.16, 1, 0.3, 1);   /* 리빌 */
--bezier-back:     cubic-bezier(0.34, 1.56, 0.64, 1);
--bezier-ambient:  cubic-bezier(0.6, 0, 0, 0.6);    /* 배경 순환 */
```
- **프레스 스쿼시(전 버튼·칩)**: 눌림 120ms `scale(0.94~0.96)` + TDS pressed 딤 → 릴리즈 320ms 복귀
- **리빌**: `opacity 0 + blur(12px) + translateY(24px)` → 800/900ms expo-out, 요소별 60~80ms 스태거 (화면 전환·카드 등장 공통)
- **스프링 2종(motion)**: 제자리 팝(탄성, 스케일 전용 — 체크·선택 링) / 위치 이동(오버슈트 금지 — FLIP·슬라이드·세그먼트 썸)
- **중단 가능 전환** + **퇴장 규율**(등장의 역방향) + 타이밍 토큰 스케일 강제
- 숫자 롤링: D-day·후보 수 변경에 카운트 롤링
- 컨페티 등 과한 해피 모먼트 장치는 미사용(체크 팝 + 오로라로 충분), 폴리시 데이 재량 항목

### 오로라 (일렁이는 배경) — 토스 방식 그대로

- **CSS radial-gradient 레이어 3~4장, blur 필터 금지**(그라데이션 falloff로 부드러움), **transform(translate/scale/rotate)만** 서로소 주기(12s/17s/23s) alternate 애니메이션 — 리페인트 0
- 색: 기본은 실측 3색 그대로 — `rgba(80,213,255,.25)` 하늘 / `rgba(255,220,200,.2)` 복숭아 / `rgba(200,230,255,.2)` 옅은 블루. 팔레트 조정은 폴리시 데이(7/8~) 재량 항목
- **적용 위치 3곳만**: 홈 상단 공기감 / 스캔 모먼트 / 완료 화면. 나머지는 단단한 화이트
- 여력 시: 히어로 1곳에 저해상도(240px) Canvas 2D 그라데이션 보강(스케일 업). reduced-motion 시 정적

### 이리데슨트 텍스트 (1곳만 — 완료 화면 타이틀)

- CSS `background-clip: text` + 실측 팔레트 `#7890EA → #50D5FF → #FFFFBB`, background-position 8s 시프트. DOM 텍스트 유지(접근성)
- 색수차 힌트는 opacity 0.15 이하 레이어로 절제 — "무지개"가 아니라 한색 계열 속 미세 분광

### 컬러·타이포·조형 (TDS 실측 토큰 자체 구현)

- primary `#3182F6`(blue500) / pressed `#1B64DA` / tint `#E8F3FF` / 텍스트 `#191F28`·`#4E5968`·`#8B95A1` / 보더 `#E5E8EB`·구획 `#F2F4F6` / 에러 `#F04452` / 경고 전경 `#B45309`(대비 보정)
- 타이포: TDS 스케일(타이틀 22~24 Bold, 섹션 17 Semibold, 본문 15, 캡션 13)
- 카드 radius 16, 여백·grey100으로 구획(테두리 박스 최소), 그라데이션은 순간에만
- 상태는 항상 아이콘+텍스트 병기. **라이트 모드만**(다크모드 안 만듦)

### 폰트 시스템 (1차 소스 확정)

- **Toss Product Sans** = UI·본문 전체. 공식 CDN `https://static.toss.im/tps/main.css` + `others.css`
- **Toss Display Sans Variable** = 워드마크·디스플레이 순간. 셀프호스팅 preload (v1 방식 그대로)
- 스택: `'Toss Product Sans', -apple-system, BlinkMacSystemFont, sans-serif` / `'Toss Display Sans Variable', 'Toss Display Sans', -apple-system, sans-serif`

### UX 라이팅

해요체·능동태·동사구 CTA("이 시간으로 할게요"). 감정 언어 금지. 가짜 로딩 금지 — 로딩엔 진짜 정보(스캔 모먼트 = 패턴 추출기의 실제 출력).

### 성능 예산 (철칙)

- 컴포지터 전용: transform/opacity만 애니메이션. 대면적 blur 금지
- React 렌더 예산: 셀·카드 memo, 파생 배열 useMemo, 안정 key. FLIP은 보이는 카드만
- 게이트: 스캔·FLIP·오로라 스크롤 프로파일링 프레임 드랍 0, 실기기(아이폰) 확인, 번들 경량 유지
