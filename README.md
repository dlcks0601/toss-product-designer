# toss calendar — 모두를 생각한 시간

> 회의 하나를 잡을 때, 참석자 전원의 하루를 먼저 생각하는 캘린더.

## 문제

회의 시간을 잡는 사람은 "빈 칸"만 봅니다. 그 칸 뒤에 있는 동료의 점심, 집중 시간,
연속 회의, 퇴근 직전의 피로는 보이지 않습니다. toss calendar는 20명 동료의 일주일을
읽고, **모두가 괜찮은 시간**을 이유와 함께 추천합니다. 안 되는 시간은 정직하게
안 된다고 말하고, 부분 참석은 당사자의 허락을 먼저 구합니다.

## 실행

```bash
npm install
npm run dev        # http://localhost:3000
```

```bash
npx vitest run     # 엔진·상태 유닛 테스트
npx tsc --noEmit   # 타입 체크
npm run build      # 프로덕션 빌드
```

외부 API·DB 없음 — 20인 시뮬레이션 월드가 코드에 내장되어 있습니다(`src/data/world.ts`).

## 주요 화면

| 화면 | 내용 |
| --- | --- |
| 홈 | 주간 캘린더 + 웰컴 카드 + 할 일 카드 + 초대 카드 + 알림 벨 |
| 셋업 | 회의 이름·참석자(아바타 피크)·길이·기한 — 카드 모핑 진입 |
| 스캔 모먼트 | 참석자 일정을 훑는 인터루드 (reduced-motion 시 생략 + aria-live) |
| 시간 찾기 | 모바일: 추천 리스트 / 데스크톱: 주간 캔버스 + 추천 카드, 이유 칩 |
| 결정 모먼트 | 모두가 되는 시간이 없을 때 — 정직한 3선택지(기한 완화·부분 참석·다른 주) |
| 확정 | 회의실 추천(인원·층 근접)·선택 → 확정 |
| 완료 | 체크 + 이리데슨트 모먼트 + "함께 챙긴 것" 요약 |
| 초대받은 사람(B) | 이유 3줄·내 행 강조·참석 응답 → 홈 고스트 블록이 실제 블록으로 |

## 기술

Next.js (App Router) · React · TypeScript · Tailwind CSS v4 · Motion · Vitest.
스케줄링 엔진은 8단계 파이프라인(`src/lib/scheduler.ts`)으로 하드 제약 → 소프트
배려 점수 → 이유 생성 순서로 후보를 만듭니다.
